# 管理台并发与日志体验收口设计

## 目标

把当前内部版后台收口成更适合多人日常使用的形态，避免页面过长、并发口径重复、日志操作不够实用。

## 设计结论

### 1. 并发口径

- 设置页只展示一个值：`可用并发`
- `可用并发 = min(系统总并发上限, 当前账号并发上限)`
- 不再对普通使用者单独展示“队列并发上限”和“我的账号并发”
- 新创建账号的默认并发改为 `5`

### 2. 账号管理页结构

- `账号管理` 页面改为两个页签：
  - `账号管理`
  - `运行日志`
- 页面顶部保留当前登录账号信息
- 默认进入 `账号管理` 页签

### 3. 账号管理交互

- 左侧保留新建账号表单，但更紧凑
- 右侧账号列表改为紧凑卡片
- 每个账号默认只展示：
  - 显示名
  - 用户名
  - 角色
  - 状态
  - 并发值
- 点击“展开”后再显示：
  - 并发调整
  - 重置密码
  - 删除账号
  - 启用/禁用
- 同一时刻只展开一个账号项，降低列表长度

### 4. 运行日志交互

- 运行日志放入单独页签
- 保留现有筛选：
  - 功能
  - 人员
  - 结果
- 补两个常用动作：
  - 导出当前筛选结果
  - 清理当前筛选结果
- 导出优先做 CSV
- 清理动作仅管理员可见，需要二次确认
- 清理后刷新列表，并给出成功提示

### 5. 服务端能力

- 保持现有写日志逻辑不变
- 新增删除日志接口，支持按筛选条件删除
- MySQL 和本地 JSON 模式都支持相同删除条件

## 范围边界

- 本轮不做日志分页
- 本轮不做批量账号操作
- 本轮不改动任务执行逻辑，只调整后台管理体验与并发默认值
