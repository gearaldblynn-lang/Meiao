# 首图子功能设计

## 背景

一键主详当前包含主图、详情、SKU 三个子功能。需要新增“首图”子功能，用于制作商品主图第一张，并为后续爆款首图裂变能力预留独立入口。首图本轮先复刻主图功能，后续再单独调整风格裂变、场景裂变等细节。

## 范围

本次新增首图作为一键主详下的独立子功能：

- 新增“首图”tab，建议排序为首图、主图、详情、SKU。
- 首图拥有独立持久化状态，不复用主图的素材、配置或方案列表。
- 首图初始功能复刻主图，包括素材上传、LOGO、设计参考、参考分析、方案策划、选择、生成、重做、同步结果、单图下载、批量下载和清空配置。
- 首图默认生成数量为 1 张。
- 除默认数量和文案标识外，其它主图能力本轮不做产品逻辑调整。

不在本次实现：

- 不新增风格裂变、场景裂变、爆款模板等专属逻辑。
- 不调整主图、详情、SKU 的现有行为。
- 不重新设计一键主详整体视觉结构。

## 状态模型

`OneClickPersistentState` 新增 `firstImage` 字段，结构与 `mainImage` 保持一致，便于本轮复刻和后续分化。

默认值与主图一致，差异如下：

- `config.count` 为 `1`。
- 其它配置保留主图默认值，例如方图、800x800、自定义尺寸、默认模型和质量。

恢复旧状态时，如果历史数据没有 `firstImage`，使用默认首图状态补齐，避免旧用户打开页面时报错。

## 组件结构

新增 `FirstImageSubModule`，初始实现复刻 `MainImageSubModule`。

`OneClickModule` 新增：

- `updateFirstImageState`
- `handleClearFirstImageConfig`
- 首图子模块挂载分支

`ConfigSidebar` 和 `SkuSidebar` 的子功能切换项新增“首图”。

## 文案与日志

首图相关 UI 文案、toast、确认弹窗、日志 action、下载文件名从“主图”替换为“首图”。

内部标识使用：

- `OneClickSubMode.FIRST_IMAGE`
- 字符串值 `first_image`

这能让后续统计和专属功能分化不污染主图数据。

## 同步策略

本轮不新增首图与主图/详情之间的同步关系。首图侧栏可以保留清空能力，但不提供同步详情或同步主图按钮，避免用户误以为首图和主图共享配置。

如果后续需要，可单独设计“从主图复制到首图”或“从首图复制到主图”的显式动作。

## 测试

新增或更新现有源码级行为测试，覆盖：

- `OneClickSubMode` 包含 `FIRST_IMAGE`。
- 默认一键主详状态包含 `firstImage`，且 `firstImage.config.count` 为 `1`。
- 恢复旧持久化状态时会补齐 `firstImage`。
- 一键主详工作区只挂载当前激活的首图子模块。
- 侧栏 tab 包含“首图”。
- 首图组件保留主图核心行为入口，例如策划、生成、重做、下载、取消策划和参考分析。

## 验收标准

- 进入一键主详后可以切换到“首图”。
- 首图默认生成数量为 1。
- 首图上传素材、策划、生成、下载等流程与主图一致。
- 清空首图只影响首图状态。
- 主图、详情、SKU 原有入口和行为不受影响。
