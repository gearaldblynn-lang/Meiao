# Dreamina 视频生成接入设计

**目标：** 将短视频工作台从壳状态接成 Dreamina CLI 的真实产品化流程，支持 `image2video`、`multiframe2video`、`multimodal2video` 三种模式，共用一个输入区、一次提交只生成一个项目卡，并能在本地和云上稳定运行。

**范围：**
- 仅改短视频模块与其后端任务链路
- 复用现有项目卡、任务队列、下载和重试能力
- 不改成独立新页面
- 不引入 text2video

**输入区体验：**
- 顶部为模式切换，三个模式共用同一个输入区
- 视觉参考借鉴即梦的轻量布局，但整体风格保持当前产品的卡片式系统
- 每次提交只创建一个项目卡，卡内承载这次提交的全部结果

**后端流程：**
- 前端把本地素材先走现有素材上传链路，拿到公网 URL 后再进入任务提交
- 后端根据模式组装 Dreamina CLI 命令
- 任务提交后进入内部任务系统，保存 providerTaskId、状态、结果和错误
- 通过 `query_result` 做轮询回写，必要时支持恢复查询

**模式差异：**
- `image2video`：单张主图 + prompt，比例从输入图推断
- `multiframe2video`：多张图，按帧间过渡语义生成连贯视频
- `multimodal2video`：图 / 视频 / 音频混合输入，支持更强参考关系

**结果页：**
- 仍使用当前项目卡风格
- 单次提交一个项目卡，卡内展示状态、结果、下载、重试、删除
- 失败任务可重试，成功任务可下载

**可靠性：**
- 不改 macOS `HOME`，避免钥匙串失效
- 所有 Dreamina 状态和结果显示都走后端，不直接依赖前端本地状态
- 所有上传素材在入模前统一保证是公网可读地址

**测试策略：**
- 后端 CLI 适配器单测覆盖 help 解析、命令参数映射、结果解析
- 后端任务路由单测覆盖三种模式的入参和回写
- 前端构建验证输入区模式切换、项目卡状态和下载入口
