# 智能体中心管理端重构设计

## 目标

把当前智能体中心管理端从“单页堆叠式后台”重构为更接近飞书/成熟 SaaS 的资源管理体验，保证：

- 浏览、创建、编辑、验证、发布、回滚的流程清楚
- 智能体与知识库边界清晰，但都仍属于智能体中心
- 页面切换自然，支持返回，不丢上下文
- 视觉更简约，减少大卡片堆叠和信息轰炸
- 第一轮先把管理端做到可用、无明显逻辑 bug，再为第二轮补充高级功能预留结构

## 信息架构

智能体中心管理端改为两大一级视图：

1. 智能体
2. 知识库

不再使用“智能体 / 知识库 / 测试台 / 统计”并列堆功能的方式。测试、版本、统计都作为智能体详情页内部的二级页面。

### 智能体一级视图

智能体部分包含三个页面状态：

1. 智能体列表页
2. 智能体详情页
3. 智能体创建/编辑向导

#### 智能体列表页

用途：

- 浏览当前可管理的智能体
- 搜索、筛选、排序
- 进入详情页
- 触发“新建智能体”

布局：

- 顶部工具条：搜索、部门筛选、状态筛选、主按钮“新建智能体”
- 主体为列表行，不用大卡片
- 每行展示：
  - 图标/首字母标识
  - 名称
  - 部门
  - 当前发布版本
  - 草稿状态
  - 验证状态
  - 最近更新时间
  - 进入详情动作

#### 智能体详情页

用途：

- 围绕“一个智能体”完成全部管理动作
- 所有相关功能都留在智能体功能里，但通过页内切换组织

布局：

- 顶部：返回列表、名称、图标、状态、负责人、主操作
- 中部：二级页签
  - 配置
  - 知识库
  - 测试
  - 版本
  - 统计

二级页签定义：

- 配置：展示当前发布版与当前草稿版的关键配置，只读或进入编辑向导
- 知识库：只负责查看和绑定已有知识库，不直接编辑知识库内容
- 测试：对当前草稿版做验证，展示可读结果摘要
- 版本：展示发布版、草稿版、最近两个历史版本，支持回滚
- 统计：展示该智能体维度的调用情况

#### 智能体创建/编辑向导

用途：

- 新建智能体
- 基于当前智能体的草稿版本做编辑

入口：

- 列表页“新建智能体”
- 详情页“编辑草稿”
- 详情页“新建草稿”

采用 5 步向导：

1. 基础信息
2. 角色与提示词
3. 绑定知识库
4. 模型与策略
5. 检查并创建/保存

规则：

- 已发布版本不可直接改，必须先新建草稿
- 编辑过程支持上一步/下一步/取消返回
- 返回详情页时保持当前智能体上下文

### 知识库一级视图

知识库部分包含两个页面状态：

1. 知识库列表页
2. 知识库创建/编辑页

#### 知识库列表页

用途：

- 浏览、搜索、筛选知识库
- 查看文档数量、绑定智能体数、更新时间
- 进入知识库编辑页
- 新建知识库

#### 知识库创建/编辑页

用途：

- 新建知识库
- 编辑知识库基础信息
- 导入文本文件或手动粘贴
- 查看、删除文档

规则：

- 智能体详情页只做“绑定已有知识库”
- 知识库内容维护只在知识库编辑页完成

## 交互原则

### 1. 单对象上下文固定

一旦进入某个智能体详情页，后续的配置、测试、版本、统计都基于同一个智能体，不再要求用户反复选择。

### 2. 浏览与创建分离

列表页负责浏览，向导页负责创建或编辑，不在详情页直接堆大表单。

### 3. 高风险动作收口

发布、回滚、归档只出现在详情页头部或版本页，且必须带状态判断与确认提示。

### 4. JSON 不直接暴露给管理者

验证结果改为结构化摘要：

- 验证是否通过
- 使用模型
- 是否检索知识库
- 命中文档
- Token 估算
- 预计成本
- 耗时
- 输出摘要

### 5. 页面返回明确

- 列表 -> 详情
- 详情 -> 列表
- 详情 -> 编辑向导 -> 保存后回详情
- 列表 -> 新建向导 -> 创建后回详情
- 知识库列表 -> 知识库编辑 -> 返回知识库列表

## 视觉方向

- 用列表行替代大量块状卡片
- 页面留白增加，信息按层级出现
- 主按钮数量严格收敛
- 尽量让一个页面只表达一件事
- 状态使用轻量标签，而不是强烈色块堆满页面

## 技术落地策略

- 保留 `modules/AgentCenter/AgentCenterModule.tsx` 入口
- 拆出管理端子组件，避免继续把所有逻辑堆在一个文件
- 继续复用现有后端接口，不做接口重构
- 优先做前端状态机和页面结构重构
- 保持本地模式与云上接口兼容

## 风险与规避

- 当前管理端状态过于集中，重构时容易选中态错乱
  - 规避：用显式页面状态和当前对象 id 管理
- 编辑草稿与已发布版容易混淆
  - 规避：详情页顶部固定展示“发布版 / 草稿版”
- 列表切换后重复请求过多
  - 规避：统一加载入口，减少 `useEffect` 互相触发

## 验收标准

- 管理员进入智能体中心后默认看到简洁的智能体列表页
- 新建智能体走向导，不再直接堆表单
- 进入智能体详情后可在页内切换配置、知识库、测试、版本、统计
- 知识库维护独立成页面，智能体里只绑定已有知识库
- 发布、回滚、归档的按钮状态与逻辑准确
- 验证结果为人可读，不再直接输出原始 JSON
- 页面切换和返回路径稳定，没有上下文丢失
