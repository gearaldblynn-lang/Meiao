# 智能体版本级知识库文档启用配置设计

## 目标

在保留“知识库统一存放文档”的前提下，让不同智能体版本可以对同一个知识库中的文档做独立启用配置。

最终效果：

- 知识库中的文档继续集中维护，不做全局启停
- 智能体版本在绑定知识库后，可以进一步勾选该知识库下哪些文档参与当前版本检索
- 工作室训练、工作室测试、正式聊天统一遵守相同的文档过滤规则
- 停用文档不删除、不重新切片，只是在检索阶段排除

## 非目标

- 不做“知识库文档全局启停”开关
- 不改变知识库文档切片、规范整理、删除逻辑
- 不做跨版本自动同步策略；配置始终以当前智能体版本为准

## 数据模型

在 `AgentVersion` 上新增版本级文档绑定配置，语义为“当前版本在每个已绑定知识库下启用了哪些文档”。

概念结构：

```json
[
  {
    "knowledgeBaseId": "kb_xxx",
    "enabledDocumentIds": ["doc_a", "doc_b"]
  }
]
```

规则：

- 仅对当前版本已绑定的知识库生效
- 某知识库未写入配置时，视为该知识库下全部文档启用，保证兼容旧数据
- 某知识库写入配置但 `enabledDocumentIds` 为空时，视为该知识库当前不提供检索内容
- 文档被删除后，残留的 `documentId` 在运行时自动忽略
- 新增文档时，旧版本若该知识库未做过显式配置，则默认可检索；若该知识库已有显式配置，则新增文档默认未启用，必须手动勾选

## 管理界面

配置入口放在智能体版本配置中，而不是知识库管理页。

交互设计：

- 保留现有“绑定知识库”选择
- 每个已绑定知识库下增加“文档启用设置”区域
- 展开后列出该知识库全部文档
- 每篇文档显示：
  - 标题
  - 切片策略
  - 片段数
  - 复选框启用状态
- 提供“全选 / 全不选”快捷操作
- 若某绑定知识库未启用任何文档，显示提示“该知识库当前不会提供检索内容”

界面语义：

- 勾选表示“当前智能体版本允许检索这篇文档”
- 取消勾选仅影响当前智能体版本，不影响知识库原文档，也不影响其他智能体版本

## 检索与运行时规则

统一新增一层“版本文档过滤”规则。

运行时流程：

1. 读取当前智能体版本绑定的 `knowledgeBaseIds`
2. 读取该版本的文档启用配置
3. 对每个知识库计算允许参与检索的 `documentId` 集合
4. 检索 SQL / 本地 JSON 均只返回这些 `documentId` 对应的片段

适用范围：

- 正式聊天
- 工作室测试
- 工作室训练

如果某知识库没有显式文档绑定配置，则保持现有行为，相当于全部文档可检索。

## 兼容与迁移

旧版本兼容规则：

- 旧数据没有文档绑定配置时，不改变现有检索结果
- 保存智能体版本时，只有用户对文档勾选做过操作，才写入显式绑定配置

迁移要求：

- MySQL 模式新增 `agent_versions` 对应 JSON 字段
- Local JSON 模式在版本对象中补充该字段
- 读取时统一做归一化，剔除不存在的知识库和文档 ID

## 接口变化

需要在智能体版本读写接口中补充该字段：

- 创建草稿版本
- 更新版本配置
- 获取版本详情
- 本地模式与 MySQL 模式保持一致

知识库详情接口无需改语义，只需继续返回完整文档列表供智能体配置界面展示。

## 测试策略

需要覆盖以下行为：

1. 旧版本无配置时，检索仍包含绑定知识库全部文档
2. 配置某知识库仅启用部分文档后，检索只返回对应文档片段
3. 两个智能体版本绑定同一知识库但启用不同文档时，检索结果彼此隔离
4. 某知识库全部取消勾选后，该知识库不再提供检索内容
5. 文档删除后，版本配置中的残留 `documentId` 不导致报错
6. MySQL 模式与 Local JSON 模式行为一致

## 风险与约束

- 智能体配置页会增加一层文档级选择，需控制展开密度，避免界面过重
- 若知识库文档很多，后续可能需要搜索或折叠；本次先不做
- “显式配置后新增文档默认不启用”是为了保持测试可控性，避免新文档自动污染既有智能体结果

## 实施范围

本次只实现：

- 版本级文档启用配置的数据结构
- 智能体配置页的文档勾选 UI
- 检索时按版本过滤文档片段
- 基础回归测试

不包含：

- 文档搜索
- 批量跨智能体复制文档配置
- 文档级权限系统
