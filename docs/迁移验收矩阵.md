# MEIAO 新壳迁移验收矩阵

更新日期：2026-05-22

## 纠偏结论

- live 入口已切回 Kimi 新壳交互：`src/main.tsx` 启动 `src/ShellMigratedApp.tsx`，并使用新壳 `SidebarNavigation`、`LandingPage`、`BottomInputBar`、`ProjectListView`、`ProjectCard`。
- 旧顶层前端目录已完成收口：运行源码统一在 `src/` 下，根目录 `components/`、`modules/`、`services/`、`utils/`、`config/`、`App.tsx`、`index.tsx`、`types.ts` 已移除。
- 旧页面组件不再作为模块主页面渲染；浏览器 DOM 不应出现旧入口链路，构建入口只允许从 `src/main.tsx` 进入 `src/ShellMigratedApp.tsx`。
- 旧稳定项目仍只读对照；本次修改全部发生在隔离项目 `版本管理/梅奥MEIAO-前端壳迁移版`。
- 新壳动作通过 `src/adapters/shellWorkflow.ts` 接旧服务层：素材上传走 `/api/assets/upload-stream`，图片生成走 `processWithKieAi -> createInternalJob/waitForInternalJob`，视频生成走 `createSoraVideoTask`。
- 新壳启动已接共享数据水合：`src/ShellMigratedApp.tsx` 优先读取 `/api/state`，经 `src/adapters/shellDataAdapter.ts` 映射为新壳项目和素材；`/api/jobs` 只用于排队/运行/待重试任务面板，不再把已完成/失败的历史任务复活为项目；失败时才回退当前 origin 的 localStorage。
- 登录态已改为真实校验 `/api/auth/me`。注意 `localhost:3000` 与 `localhost:3001` 是不同浏览器 origin，前端 token/localStorage 不自动共享；3001 登录同一后端账号后，共享的是 3100 后端状态、任务和日志数据。
- 新壳已按“模块 + 子功能”隔离对话区、素材区、项目区和任务区；一键主详的首图/主图/详情页/SKU、翻译的主图出海/详情出海/去文案等不会互相混列表。本轮额外修复一键主详旧后端任务污染：没有显式 `subFeature` 的 `kie_image` 会从 prompt 中的首图/主图/详情页/SKU 语义反推，无法识别的旧任务进入 `legacy_unassigned`，不再默认塞进主图。
- Kimi 壳子原有示例方案和示例预设已移除；项目展示改为卡片网格入口，支持搜索、按日期和状态筛选，点击卡片进入项目详情。
- 项目详情已恢复通用深层管理入口：结果预览、生图 Prompt 展示/复制、单图下载、批量打包下载、单结果删除、项目删除、找回同步、重新生成。详情弹窗宽度节奏对齐账号管理页，主体改为响应式结果卡对照：宽屏三张并排、中屏两张并排、窄屏单张；每张结果卡把图片、模型/比例/状态、Prompt 抽屉、下载/复制/找回/重生成/删除固定在一起，不再拆成一个巨大下方面板。详情页/详情出海额外支持“单张查看 / 拼合查看”，便于长图衔接核对。
- 智能体中心已按新壳风格重建，不再包旧前端组件：智能体广场、会话、管理员工厂、知识库、版本发布/回滚/验证、调用统计、图像生成长超时与附件上传均直接接共享 3100 后端。
- 账号页已按新壳风格重建，不再包旧前端组件：管理员账号管理、运行日志筛选/分页/导出/清理、用量统计均直接接共享 3100 后端。

## 运行边界

- 当前稳定项目：`版本管理/梅奥MEIAO-当前版本`，只读对照。
- 新壳迁移项目：`版本管理/梅奥MEIAO-前端壳迁移版`。
- 新前端端口：`http://localhost:3001`。
- 共享后端：`http://127.0.0.1:3100`，新前端通过 `/api` 代理访问。
- 本地验收入口：`http://127.0.0.1:3001/?meiaoLocalPreview=1`。
- 模块直达验收入口：`http://127.0.0.1:3001/?meiaoLocalPreview=1&module=one_click` 等。

## 数据治理必做

- 前端项目展示以 `/api/state` 的账号持久化业务状态为准；用户删除项目后，该项目不应再通过 `/api/jobs`、localStorage 兜底或其它历史来源重新出现。
- 删除语义需要分层：业务项目和关联结果大文件应被清理；轻量审计日志保留“谁、何时、删除了什么、清理是否成功”，用于排查，不保留图片/视频等重资产本体。
- 迁移 3000 到 3001 时必须梳理旧数据：识别历史任务记录、有效业务项目、孤儿结果文件、已删除但仍可恢复的残留记录，并给出清理/归档结果。
- 云端控量需要后端定期清理任务：完成/失败且已不被任何业务项目引用的 `jobs`、结果文件、临时上传文件，应按保留期自动删除；排队/运行/待重试任务不得清理。
- 清理功能要可审计、可预演：正式删除前支持统计待清理数量和空间占用；执行后记录轻量日志，避免误删真实项目。

## 仍需继续补齐

- 旧项目深层能力仍需继续按模块补齐专属工作台：预设库保存/应用、白底/原图精修细分、买家秀纯文案、Veo 分段、视频诊断。
- 智能体已改为新壳原生页面；旧工作室训练的多轮配置差异应用仍需继续做成新壳风格的二级工作台，当前已覆盖聊天、附件、生图、工厂基础编辑、版本发布/回滚/验证、知识库和统计。
- 账号管理已改为新壳原生页面；个人资料头像编辑等非管理员自助项仍需继续补成新壳风格，当前已覆盖管理员账号、日志和统计。
- 通用项目级找回/重试/打包导出已补入口；其中后端 job 结果重试会调用 `/api/jobs/:id/retry`，仅有持久化快照且缺少 job 任务元信息的旧结果，会先把 Prompt 回填到底部输入区供用户按当前模块重新提交。
- 真实模型链路需要在登录内部模式后逐项跑：上传真实图片、提交任务、等待结果、取消任务、失败展示、恢复/找回，并在项目详情中核对完整 Prompt、上下拼合预览和批量下载。
