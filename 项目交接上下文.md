## 2. 用户协作偏好
- 用普通人能听懂的话说明，不要太技术化。
- 优先做最小正确方案，不做补丁式越修越乱的实现。
- 改功能时，要同步考虑日志、测试、版本管理，不只改表面功能。
- 允许边改边测试，优先保证现有可用功能不被破坏。

## 3. 当前产品定位
- 这是梅奥 AI 电商视觉工具。
- 当前主要模块：
  - 一键主图 / 一键详情
  - 出海翻译
  - 买家秀
  - 产品精修
  - 短视频
  - 设置
  - 账号管理
- 目标方向：
  - 先稳定本地版
  - 再推进公司内部多人使用版

## 4. 当前已完成的重要能力
- 已接入内部账号系统：
  - 登录 / 登出
  - 管理员创建账号
  - 启用 / 禁用 / 重置密码 / 删除账号
- 已做用户隔离：
  - 每个用户的工作台状态独立
  - 上传素材尽量按用户隔离
  - 生成结果和项目记录按用户隔离
- 已有日志系统：
  - 管理员可在账号管理页查看运行日志
  - 日志支持按功能 / 人员 / 结果 / 日期查看
  - 日志保留 7 天
  - 当前要求是：后续新增或修改功能时，日志也要同步补

## 5. 日志规则
- 日志不是记录所有 UI 变化，只记录“有业务意义的按钮动作 + 任务结果”。
- 当前统一日志帮助层文件：
  - `services/loggingService.ts`
- 当前日志展示页：
  - `modules/Account/AccountManagement.tsx`
- 后续如果新增功能，默认同步补：
  - 开始
  - 成功
  - 失败
  - 中断
  - 删除 / 清空 / 导出 / 重试 / 找回

## 6. 版本管理规则
- 本地版本文件夹命名规则：
  - `梅奥MEIAO——年月日字母`
  - 例如：
    - `梅奥MEIAO——260320a`
    - `梅奥MEIAO——260320b`
- Git 提交编号和版本文件夹名不是一个东西。
- 用户更容易记版本文件夹名，所以对外沟通优先说文件夹版本名。

## 7. 云部署方向
- 目标是做公司内部多人使用版，不是直接做公网开放 SaaS。
- 当前方向：
  - 腾讯云 / 宝塔
  - MySQL 已准备
  - 后续继续完善账号、隔离、日志、稳定性
- 当前结论：
  - 还不适合直接公开给外部用户使用
  - 更适合内部使用和联调

## 8.1 本地开发默认入口
- 本地默认开发页：
  - `http://localhost:3000`
- 本地后端健康检查：
  - `http://127.0.0.1:3100/api/health`
- 推荐本地启动命令：
  - `npm run local`
- 推荐本地排查命令：
  - `npm run doctor`
- 说明：
  - `3000` 是默认开发页入口
  - `3100` 是本地后端接口端口
  - 如果只开了 `3100` 没开 `3000`，浏览器打开 `localhost:3000` 会报连接被拒绝

## 9. 当前建议默认检查项
- 改功能时检查：
  - 是否影响现有功能
  - 是否需要补日志
  - 是否需要补用户隔离
  - 是否需要补管理员可排查信息
  - 是否需要存版本
